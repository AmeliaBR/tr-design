3. About levels

The title of a specification doesn't need to reflect its level number.

Some Groups feel strongly about the levels of their specification, others prefer not to deal with them.

Level is a number, not interpreted by the underlying system except for its sequence order.

The underlying links must have the information about the level. Not all headers reflected in this document have to be there (eg "Latest published Level 2 version" isn't necessary).

4. The meaning of Latest

"Latest" will return the most "stabilized" document available.

As such, it points to the first document in the following list:

  1. the highest level being a CR, a PR, or a REC that is not obsoleted/superceded/rescinded
  2. the highest level being a Note if not retired
  3. the lowest level being a WD
  4. the obsoleted/superceded/rescinded REC
  5. the retired Note

5. The meaning of Upcoming

"upcoming" will return the "tip" document available.

As such, it points to the first document in the following list:

  1. the latest version of the highest level not obsoleted/superceded/rescinded/retired
  2. the obsoleted/superceded/rescinded/retired document

6. The meaning of Ed

"ed" will redirect to the "nightly version" of the upcoming document if available.

As such, it redirects to the document known as "editor's draft" that is associated with the upcoming document, if known.
